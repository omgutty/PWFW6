    Interview Answer вЂ” Hybrid Testing Architecture

"Hybrid tests are the most efficient pattern in enterprise test automation. 
The principle is simple вЂ” use the fastest mechanism to set up test state,
 and use the most accurate mechanism to validate the outcome. 
 API calls are typically 10 to 20 times faster than UI interactions for the same operation.
  So for any test where the setup is not what is being tested, we use the API. 
  For the assertion вЂ” the thing we are actually validating вЂ” we use whichever layer
   gives us the most confidence. A test validating cart total would set up via API 
   and assert via UI, because the cart total is a UI concern. A test validating that
    filtering works correctly in the backend would use API for both setup and assertion.
     We also use hybrid tests for contract testing вЂ” 
     validating that API response shapes match our TypeScript interfaces,
      which acts as an early warning system for breaking backend changes 
      before they surface in UI test failures
